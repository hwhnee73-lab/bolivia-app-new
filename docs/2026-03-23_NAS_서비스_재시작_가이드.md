# NAS Bolivia App 서비스 재시작 가이드

> **작성일**: 2026-03-23 | **대상 서버**: NAS (`192.168.0.29`)

---

## 접속 정보

| 항목 | 값 |
|------|---|
| SSH 접속 | `ssh nas` (포트 4444) — **Docker 명령은 이 접속만 가능** |
| 소스 경로 | `/volume1/homes/mcp/Bolivia-app/` |
| 서비스 | `frontend` (80, 443), `backend` (8080), `db` (3306) |

> [!WARNING]
> `ssh nas-dev` (포트 2222)에서는 Docker를 사용할 수 없습니다. 반드시 `ssh nas`로 접속하세요.

---

## 1. 서비스 종료

### A. 전체 서비스 종료 (DB 데이터 유지)

```bash
ssh nas
cd /volume1/homes/mcp/Bolivia-app
docker compose down
```

### B. 개별 서비스만 종료

```bash
# 백엔드만 종료
docker compose stop backend

# 프론트엔드만 종료
docker compose stop frontend

# DB만 종료 (⚠️ 백엔드도 함께 중지 권장)
docker compose stop db
```

### C. 강제 종료 (응답 없을 때)

```bash
# 특정 컨테이너 강제 종료
docker kill bolivia-backend
docker kill bolivia-frontend

# 전체 강제 종료 후 정리
docker compose kill
docker compose down
```

### D. 종료 상태 확인

```bash
# 실행 중인 컨테이너 확인 (없으면 종료 완료)
docker ps | grep boliv

# 중지된 컨테이너 포함 전체 확인
docker ps -a | grep boliv
```

## 2. 재시작 시나리오별 명령어

### A. 백엔드만 재시작 (코드 변경 후)

```bash
ssh nas
cd /volume1/homes/mcp/Bolivia-app
docker compose up -d --build backend
```

### B. 프론트엔드만 재시작

```bash
ssh nas
cd /volume1/homes/mcp/Bolivia-app
docker compose up -d --build frontend
```

### C. 전체 서비스 재시작 (DB 데이터 유지)

```bash
ssh nas
cd /volume1/homes/mcp/Bolivia-app
docker compose down
docker compose up -d --build
```

### D. 전체 초기화 (⚠️ DB 데이터 삭제)

```bash
ssh nas
cd /volume1/homes/mcp/Bolivia-app
docker compose down -v         # 볼륨까지 삭제
docker compose up -d --build   # 재생성
```

> [!CAUTION]
> `-v` 옵션은 DB 데이터를 포함한 모든 볼륨을 삭제합니다. 운영 데이터가 있다면 **반드시 백업** 후 실행하세요.

## 3. 코드 반영 후 재시작 전체 흐름

```
[로컬] git push origin main
         ↓
[NAS] ssh nas-dev → git pull / clone + cp
         ↓
[NAS] ssh nas → docker compose up -d --build backend
         ↓
[NAS] docker logs bolivia-backend --tail=30  ← 기동 확인
```

### Step 1: 소스 동기화 (nas-dev에서)

```bash
ssh nas-dev
cd ~/nas/Bolivia-app

# 방법 1: git pull (직접)
git pull origin main

# 방법 2: clone + cp (권한 문제 시)
git clone --depth 1 https://github.com/hwhnee73-lab/bolivia-app-new.git /tmp/bolivia-new
sudo cp -r /tmp/bolivia-new/backend ~/nas/Bolivia-app/
sudo cp -r /tmp/bolivia-new/frontend ~/nas/Bolivia-app/
sudo cp /tmp/bolivia-new/docker-compose.yml ~/nas/Bolivia-app/
rm -rf /tmp/bolivia-new
```

### Step 2: Docker 재빌드 (nas에서)

```bash
ssh nas
cd /volume1/homes/mcp/Bolivia-app
docker compose up -d --build
```

> 💡 NAS에서 빌드는 **10~20분** 소요될 수 있습니다.

## 4. 기동 확인

```bash
# 컨테이너 상태 확인
docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}" | grep boliv

# 백엔드 로그 확인 (Started 메시지 확인)
docker logs bolivia-backend --tail=20

# 백엔드 API 헬스 체크
curl -s http://localhost:8080/api/health

# 프론트엔드 확인
curl -s http://localhost | head -c 100
```

## 6. 트러블슈팅

| 증상 | 원인 | 해결 |
|------|------|------|
| `docker: command not found` | `nas-dev`(2222)로 접속함 | `ssh nas`(4444)로 접속 |
| `no configuration file provided` | 잘못된 디렉터리 | `cd /volume1/homes/mcp/Bolivia-app` |
| Hibernate 스키마 에러 | Entity ↔ DB 컬럼 타입 불일치 | Entity의 `columnDefinition` 수정 |
| DB Unhealthy | NAS 디스크 I/O 느림 | healthcheck `start_period: 120s` 조정 |
| `Permission denied` | NAS 볼륨 UID 불일치 | `sudo cp -r ...` 사용 |
| 빌드 오래 걸림 (20분+) | NAS CPU/IO 한계 | `--no-cache` 제거, 캐시 활용 |
| `.env` 누락 | git에 포함 안 됨 | 수동 생성 (배포 가이드 Step 3 참조) |
| 백엔드 즉시 종료 | `.env` 미설정 또는 DB 미기동 | `.env` 확인 + `docker ps`로 DB 상태 확인 |

### DB 컨테이너가 unhealthy일 때

```bash
# healthcheck 시간 늘리기
sed -i 's/start_period: 20s/start_period: 120s/' docker-compose.yml
sed -i 's/retries: 30/retries: 60/' docker-compose.yml
docker compose down && docker compose up -d --build
```
